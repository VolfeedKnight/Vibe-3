# 공공직군 행정업무 슈퍼앱 PRD

## 1. 목적

공공직군 행정 담당자가 반복적으로 수행하는 일정 공유, 엑셀 처리, 민원 대응, 행정 뉴스 확인 업무를 하나의 웹 애플리케이션에서 처리할 수 있도록 한다.

이 문서는 구현 전 단계의 기능 요구사항을 정의한다.

## 2. 대상 사용자

- 공공기관 행정 담당자
- 부서 관리자
- 팀원 일정과 업무 현황을 확인해야 하는 실무자
- 민원 응대 매뉴얼을 참고해 답변 초안을 작성하는 담당자

## 3. 핵심 기능

### 3.1 팀원 스케쥴 관리

팀원들의 휴가, 근무, 출장 일정을 공유하는 캘린더 기능을 제공한다.

#### 주요 요구사항

- 월간, 주간, 일간 캘린더 화면을 제공한다.
- 일정 유형은 최소 `휴가`, `근무`, `출장`, `기타`를 지원한다.
- 일정에는 제목, 대상자, 시작일시, 종료일시, 장소, 메모를 기록할 수 있다.
- 일정 유형별 색상을 구분한다.
- 팀원별, 일정 유형별 필터를 제공한다.
- 일정 등록, 수정, 삭제 기능을 제공한다.
- 같은 시간대 일정 중복 여부를 화면에서 확인할 수 있어야 한다.

#### 우선순위

- 1차: 일정 CRUD, 월간 캘린더, 유형별 색상
- 2차: 팀원/유형 필터, 중복 일정 표시
- 3차: 알림, 반복 일정, 권한별 수정 제한

### 3.2 엑셀 업무 자동화  :  Python - Openpyxl, Pandas

특정 컬럼을 기준으로 엑셀 파일을 나누거나 여러 엑셀 파일을 하나로 합치는 기능을 제공한다.

#### 주요 요구사항

- `.xlsx` 파일 업로드를 지원한다.
- 사용자가 기준 컬럼을 선택할 수 있어야 한다.
- 기준 컬럼의 값별로 파일을 분리해 다운로드할 수 있어야 한다.
- 여러 엑셀 파일을 업로드해 하나의 파일로 병합할 수 있어야 한다.
- 병합 시 컬럼 구조가 다르면 사용자에게 경고한다.
- 처리 결과 파일은 다운로드 가능한 형태로 제공한다.
- 대용량 파일 처리 시 진행 상태를 표시한다.

#### 우선순위

- 1차: 단일 파일 분리, 다중 파일 병합
- 2차: 컬럼 불일치 검증, 처리 결과 미리보기
- 3차: 처리 이력 저장, 예약 처리

### 3.3 민원 대응 챗봇   : AI - OPENAI API,   Python - Langchain + RAG

첨부된 민원 매뉴얼을 기반으로 민원 대응 방식과 답변 스크립트 초안을 작성하는 챗봇 기능을 제공한다.

#### 주요 요구사항

- 민원 매뉴얼 파일 업로드를 지원한다.
- 사용자는 민원 내용 또는 질의 내용을 입력할 수 있다.
- 챗봇은 관련 매뉴얼 내용을 참고해 대응 방향과 답변 초안을 제시한다.
- 답변에는 근거가 된 매뉴얼 문서명 또는 섹션을 함께 표시한다.
- 민감정보 입력 방지를 위한 안내 문구를 제공한다.
- 생성된 답변은 사용자가 최종 검토 후 사용할 수 있음을 명확히 표시한다.

#### 우선순위

- 1차: 매뉴얼 업로드, 질의 입력, 답변 초안 생성 UI
- 2차: 근거 문서 표시, 답변 템플릿 선택
- 3차: 부서별 매뉴얼 관리, 대화 이력 저장

### 3.4 뉴스 기사 수집    : Python - BS4 (Selinium 현재는 필요없음)

공공 행정 관련 뉴스를 매일 아침 수집해 한 화면에서 확인할 수 있도록 한다.

#### 주요 요구사항

- 공공 행정 관련 키워드를 기준으로 뉴스 기사를 수집한다.
- 매일 아침 정해진 시간에 자동 수집한다.
- 기사 제목, 언론사, 발행일시, 요약, 원문 링크를 제공한다.
- 키워드별 필터와 검색 기능을 제공한다.
- 중복 기사 제거 로직을 적용한다.
- 사용자가 관심 키워드를 관리할 수 있어야 한다.

#### 우선순위

- 1차: 수동 수집, 기사 목록 표시
- 2차: 매일 아침 자동 수집, 키워드 관리
- 3차: 요약, 중요도 분류, 이메일 또는 메신저 알림

## 4. 비기능 요구사항

### 4.1 보안

- 민원 내용과 매뉴얼에는 개인정보가 포함될 수 있으므로 입력/저장 정책을 명확히 한다.
- 사용자 인증과 권한 관리를 고려한다.
- 파일 업로드 시 확장자, 크기, 악성 파일 여부를 검증한다.
- 외부 API를 사용하는 경우 API Key는 서버 환경변수로 관리한다.

### 4.2 성능

- 일반 일정 조회는 1초 이내 응답을 목표로 한다.
- 엑셀 파일 처리는 파일 크기에 따라 비동기 처리 구조를 고려한다.
- 뉴스 수집 작업은 사용자 요청 처리와 분리된 백그라운드 작업으로 설계한다.

### 4.3 접근성

- 공공 업무 환경을 고려해 명확한 텍스트, 충분한 대비, 키보드 탐색을 지원한다.
- 캘린더 색상은 색상만으로 의미를 구분하지 않고 라벨을 함께 제공한다.

### 4.4 감사 및 이력

- 일정 변경, 파일 처리, 챗봇 사용, 뉴스 수집 실행 이력을 남길 수 있도록 설계한다.
- 민원 답변은 자동 발송하지 않고 사람이 최종 검토하는 흐름을 기본으로 한다.

## 5. 데이터 개요

### 주요 엔티티

- User: 사용자 계정
- TeamMember: 팀원 정보
- Schedule: 일정 정보
- ExcelJob: 엑셀 처리 작업
- ManualDocument: 민원 매뉴얼 문서
- ChatSession: 챗봇 대화 세션
- NewsArticle: 수집된 뉴스 기사
- NewsKeyword: 뉴스 수집 키워드

## 6. MVP 범위

MVP는 다음 기능까지 포함한다.

- 팀원 일정 등록, 조회, 수정, 삭제
- 엑셀 파일 컬럼 기준 분리
- 엑셀 파일 병합
- 민원 매뉴얼 업로드와 답변 초안 생성 화면
- 공공 행정 뉴스 수동 수집과 목록 표시

## 7. 제외 범위

초기 버전에서는 다음 기능을 제외한다.

- 기관 SSO 연동
- 전자결재 시스템 연동
- 실제 민원 자동 발송
- 모바일 앱
- 고급 권한 체계
- 대규모 문서 검색 인프라

## 8. 성공 기준

- 사용자가 하나의 화면에서 일정, 엑셀 처리, 챗봇, 뉴스 기능으로 이동할 수 있다.
- 행정 담당자가 반복 엑셀 작업을 수동 작업 대비 줄일 수 있다.
- 민원 답변 초안 작성 시간을 단축할 수 있다.
- 매일 확인해야 하는 행정 관련 뉴스를 한 곳에서 확인할 수 있다.
