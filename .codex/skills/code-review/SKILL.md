# 코드 리뷰

## 개요

FE와 BE 개발이 끝난 뒤 변경 내용을 검토한다. 사용자가 명시적으로 코드 리뷰를 요청한 경우에만 사용한다.

## 리뷰 절차

1. 변경 범위와 의도를 파악한다.
2. 직관성을 점검한다.
3. 보안취약성을 점검한다.
4. 재사용성을 점검한다.
5. 결과를 한국어 표로 정리한다.

## 점검 기준

### BE : uv 기반 python + fast api

- 입력 검증과 에러 처리가 명확한지 확인한다.
- 인증, 인가, 비밀정보 노출, 주입 취약성, 경로 조작 가능성을 확인한다.
- 비동기 처리와 I/O 경계가 안전한지 확인한다.
- 공통 로직을 재사용 가능한 함수, 의존성, 서비스로 분리할 수 있는지 본다.

### FE: React + TypeScript + Vite

- 컴포넌트 이름, 책임, 상태 흐름이 직관적인지 확인한다.
- XSS, 안전하지 않은 렌더링, 외부 입력 처리 문제를 확인한다.
- 훅 의존성, 타입 안정성, 예외 처리, 비동기 상태 관리가 자연스러운지 확인한다.
- 반복되는 UI나 로직을 재사용 가능한 컴포넌트, 훅, 유틸로 묶을 수 있는지 본다.

## 결과 형식

- 반드시 표 형식으로 제시한다.
- 권장 열: `구분`, `위치`, `문제`, `영향`, `권장 조치`, `심각도`.
- 발견 사항이 없으면 그 사실도 표로 보여준다.
- 각 항목은 간결하게 쓰고, 실제 수정이 필요한 부분을 우선순위로 정리한다.
